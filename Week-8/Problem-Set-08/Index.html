Topics: Client-Side Routing with React Router

Problem A: Code Changes

  • Wrapped <App> in <BrowserRouter> in main.jsx to enable client-side routing
  • Set up <Routes> with paths for /about, /resources, and /adopt
  • Created nested routes under /adopt with an index route for <PetList> and a :petName param route for <PetDetail>
  • Added <Outlet /> in PetPage to render child route content
  • Used <Navigate> catch-all route to redirect unknown URLs to /adopt
  • Replaced <a> tags with <NavLink> in navigation and <Link> for pet cards and header
  • Used useParams() hook in PetDetail to access the pet name from the URL

Key Concepts

  • BrowserRouter: Wraps the app to track URL state and trigger re-renders on navigation
  • Routes & Route: Maps URL paths to specific components
  • Nested Routes: Child routes render inside a parent's <Outlet />
  • Index Route: Default child route rendered when the parent path is matched exactly
  • URL Parameters: :param syntax in paths, accessed with useParams()
  • Link & NavLink: Client-side navigation without page reloads; NavLink adds active styling
  • Navigate: Programmatic redirect component for catch-all routes
